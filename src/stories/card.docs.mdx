# Card

A flexible container component for grouping related content with composable sub-components.

## Usage

```tsx
import {
  Card,
  CardHeader,
  CardTitle,
  CardDescription,
  CardContent,
  CardFooter,
} from '@tc96/ui-react'

function Example() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>Card Title</CardTitle>
        <CardDescription>Card description goes here</CardDescription>
      </CardHeader>
      <CardContent>
        <p>Your content here</p>
      </CardContent>
    </Card>
  )
}
```

## Sub-Components

- **Card**: The root container
- **CardHeader**: Top section for title and description
- **CardTitle**: Heading element for the card title
- **CardDescription**: Subtitle or descriptive text
- **CardContent**: Main content area with padding
- **CardFooter**: Bottom section, often for actions

## Accessibility

- Card is semantically a `<div>` container
- For interactive cards, wrap in a `<button>` or `<a>` tag
- Use proper heading hierarchy with CardTitle
- Ensure sufficient color contrast for text
- Add hover/focus states for clickable cards

## Best Practices

✅ **Do:**
- Use consistent card sizing in grids
- Include clear titles and descriptions
- Group related information together
- Use CardFooter for actions/metadata
- Maintain consistent padding and spacing

❌ **Don't:**
- Nest cards inside cards (usually)
- Overcrowd with too much information
- Use cards for single pieces of data
- Make cards too wide (max ~600px)
- Use cards when a simpler layout works
